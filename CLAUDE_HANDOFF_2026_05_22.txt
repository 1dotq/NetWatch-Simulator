2. APPLICATION ARCHITECTURE OVERVIEW
================================================================================

The DigitalTwinApp class (app.js) is the central controller. It manages:
  - this.canvas       → NetworkTwinCanvas instance (canvas rendering + interaction)
  - this.sim          → ReactorPhysicsSim instance (thermodynamics step engine)
  - this.orchestrator → AIOrchestrator instance (threat detection + alerts)
  - this.inspector    → DeviceInspector instance (device configuration panels)
  - this.llmService   → AELlmService instance (Gemini/Claude API calls)

Main render loop:
  loop(t) → requestAnimationFrame → sim.step() → orchestrator.evaluateState()
           → annotateReactorNodes() → canvas.update() → canvas.draw()
           → sim.drawChart() → updateLinkUtilisation() → updateThreatVisuals()

Two project types exist:
  - this.activeProjectType === 'reactor'  → ICS/OT industrial plant scenario
  - this.activeProjectType === 'campus'   → Corporate IT networking scenario

Landing portal with 31 labs is shown before a project is selected.

3. DATA STRUCTURES
================================================================================

canvas.nodes — Array of node objects:
  {
    id: String,              // Unique ID (e.g. 'PLC-101', 'PE-01')
    name: String,            // Human label
    type: String,            // 'it' | 'ot' | 'plc' | 'field'
    role: String,            // 'Router' | 'Switch' | 'Firewall' | 'Modbus PLC' |
                             // 'SCADA HMI' | 'Actuator' | 'Sensor' | 'AD Server'
    ip: String,              // Static IP (e.g. '10.1.10.1')
    x: Number, y: Number,    // Canvas world coordinates
    status: String,          // 'stable' | 'compromised' | 'isolated' | 'offline'
    ospfState: String,       // 'N/A' | 'DOWN' | 'INIT' | 'FULL'
    firmware: String,        // Version string
    note: String,            // Optional user sticky note
    pinned: Boolean,         // Pinned prevents drag (IMP-28 context menu)
    _liveAnnotation: String, // Live sim value badge (IMP-49, reactor nodes)
    _searchMatch: Boolean,   // Search filter dim state (IMP-31)
  }

canvas.links — Array of link objects:
  {
    sourceId: String,
    targetId: String,
    sourceInterface: String, // e.g. 'Gig0/1'
    targetInterface: String,
    status: String,          // 'normal' | 'offline' | 'isolated'
    speed: Number,           // Mbps 1-100, fluctuates via updateLinkUtilisation()
    encrypted: Boolean,      // Shows green glow (IMP-10)
    protocol: String,        // Auto-annotated: 'Modbus/TCP' | 'OSPF/BGP' | etc. (IMP-41)
  }

ReactorPhysicsSim state:
  this.level    — Liquid level % (0-100)
  this.pressure — Internal pressure MPa (0-4.0)
  this.temp     — Reactor temperature °C (15-130)
  this.inletValve  — V-101 opening % (0-100)
  this.outletValve — V-102 opening % (0-100)
  this.reliefValve — XV-103 boolean (vent valve)
  this.sisInterlock — Safety Instrumented System interlock enabled
  this.peakPressure / this.peakTemp — Running max values (IMP-1)
  this.alarmTriggered — Critical threshold flag (IMP-2)

  WHAT WAS DONE:
  Replaced the generic Cisco IOS CLI help menus for OT/SCADA devices with

  DEVICES WITH CUSTOM CLI:

      show plc status | show io-map | write register <addr> <val>

      show valve position | open | close | set position <0-100>

      show measurement | show range | set range <low> <high>
      show status | zero-calibrate | show diagnostics
      set alarm-high <val> | set alarm-low <val> | show history

      show sis status | show safety-loop | trip | reset
      show diagnostic | set bypass <on|off> | show audit-log

      show drive status | set frequency <hz> | start | stop
